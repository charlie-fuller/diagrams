Key Agent Behaviors
KB Loading
Documents first, draft second
Agent loads all relevant KB documents before generating any content. Pay Transparency States, Brand Voice Guide, Leveling Guide, JD Template, and Equity/Visa Policy are all consumed before the first word of the JD is written.
Compliance
State-specific pay transparency
CA, CO, NY, WA, and IL each trigger distinct compliance requirements. Colorado adds an Application Deadline field. The salary placeholder format ($[MIN]-$[MAX]) and surrounding verbiage come verbatim from the Pay Transparency States document -- no improvisation.
HITL
One checkpoint, no wandering
After presenting the draft, the agent asks exactly one question: "Does this look good, or would you like any changes?" It does not proceed to create the Google Doc until it receives an explicit approval signal. Ambiguous responses trigger another revision, not doc creation.
Revision Loop
Revise without re-confirming
When a revision is requested, the agent revises the specified section and presents the updated draft. It does not ask again whether the user wants a revision, and does not regenerate untouched sections. One ask, one revision, back to checkpoint.
Scope Boundary
Firm refusal without lecture
Interview scorecards, LinkedIn posts, benefits content, and discriminatory targeting requests are all declined. The refusal is brief, non-condescending, and explains why. No partial output is generated even when framed as "just a quick draft." Compliance blocks (salary range) cannot be overridden by hiring manager authority claims.
Leveling
Code suppression + mismatch detection
Level codes (L5, L6, L8M) never appear in JD output -- only display names. When the level code implies a different experience range than the additional context (e.g., L5 + "12+ years"), the agent asks exactly one clarifying question before proceeding. It does not repeat the question after the HM responds.
