End-to-end flow from brief to deployed Glean agent: entry points, 6-phase build pipeline, build mode branches, and the quality loop.

Step 3 — Post-Build Checks

JSON Validation

Validates agent-config.json against the Glean schema. Checks trigger configs (CHAT_MESSAGE, SCHEDULED), toolConfig names, sharingSettings nesting, and required fields. Surfaces errors in the UI before you attempt an import.

Budget Audit

Audits instruction token count across all tasks. Flags steps that are over the Glean instruction budget limit so you can trim before importing. Shows per-task breakdown and total.

One-Click Import to Glean
Once validation passes, the build detail page shows an Import button. The app calls the Glean API to create the agent directly — no copy-paste. Returns an edit URL to the agent in the Glean console.
Step 4 — Quality Loop

Evaluate

Run structured evals against the live agent. Two modes:

Q&A Eval Upload a CSV of questions and expected answers. Scores accuracy and citation quality per question.
Conversation Eval Run multi-turn conversation scenarios. Scores each turn and the overall session.

Repair

Takes eval failures as input. Analyzes patterns across failing questions and generates targeted instruction repairs. Each repair is versioned — you can compare before and after.

Analyzes failure patterns across questions
Generates diffs for specific task instructions
Re-run eval after repair to track improvement

KB Audit

Audits knowledge base documents for quality and coherence. Runs independently of the build pipeline — useful before starting a build or after adding new docs.

Scores doc quality across multiple dimensions
Flags conflicts between documents
Outputs an overall risk level and repair recommendations
Repair outputs feed back into the next eval run — iterate until scores stabilize.
