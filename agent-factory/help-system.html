Architecture Decision — Full Corpus vs RAG
Why no retrieval pipeline? Full corpus chosen
Full Corpus Loading (chosen)
  • Entire docs/help/user/ loaded into the system prompt on every call
  • No synonym blindness — the model sees exact phrasing from every doc
  • No missing-context refusals from incomplete chunk retrieval
  • Prompt caching makes it cheap after the first call (~50–80K tokens cached)
  • Works at this corpus size — help docs are intentionally small
RAG / Chunk Retrieval (not used)
  • Keyword search misses synonyms and paraphrased questions
  • Wrong chunks surface for ambiguous queries
  • Partial context causes the model to refuse or hallucinate gaps
  • Vector index is available as an admin debug tool only — not on the answer path
  • Over-engineered for a corpus this size
Prompt Structure and Caching
SYSTEM Behavior Instructions Voice, fabrication constraints, canonical "what is" answer, rubric dimensions, supported file types per-call
CACHED Full Help Corpus Every docs/help/user/*.md file concatenated into one block. Pinned with cache_control: ephemeral. Only paid once until docs change. cached
HISTORY Last 3 Exchanges The previous 6 messages (3 user + 3 assistant). Enough context for follow-ups without inflating cost. per-call
USER Current Question The new question appended as the final user message per-call
Anti-Fabrication Guardrails
UI Elements
Never name buttons, fields, toggles, or menu items that don't appear in the corpus. If the context doesn't confirm the label, don't use it.
Features and Capabilities
No describing features, behaviors, or workflows not confirmed by the retrieved context. Includes speculative "you could probably also..." phrasing.
Numerical Thresholds
No inventing scoring tiers, color-coded ranges, or numeric cutoffs. Eval rubric dimensions and weights are hard-coded in the system prompt as the canonical source.
Output Artifacts
Agent Factory has four named build outputs: Workflow Mode JSON, Auto Mode brief, Skill (SKILL.md), Custom Action (OpenAPI spec). No secondary artifacts beyond what the corpus names.
Third-Party Integrations
No speculating about integrations (Slack, webhooks, scrapers) not in the corpus. If the user asks about something uncovered, say so and suggest contacting an admin.
Legal and Ethical Edge Cases
If a question raises legal or ethical concerns (web scraping, surveillance, TOS evasion), flag the concern explicitly first. Don't decline on technical grounds alone. Direct to legal or compliance.
Self-Evaluation Loop
01
Load Help Corpus
Reads all docs/help/user/*.md as the KB source for test generation
02
Generate Test Plan
Uses the shared testplan_generator with a help-specific PRD to produce Q&A pairs from the corpus
03
Run Each Question
Calls help_service.ask() for every generated question — same code path as live user queries
04
Score Accuracy
score_accuracy() evaluates each answer. Results persisted to evaluations with reserved agent_id="__help_system__"
The eval uses the same scoring pipeline as agent evals — no separate tooling. The help system grades itself using its own infrastructure. Scores are visible in the Evaluate section filtered to the __help_system__ agent.
