Agent Factory — Functions Reference

All 5 numbered nav steps, what each one does, what it takes in, what it outputs, and how they connect to each other.

Numbered Steps
1
Discovery
Turn a fuzzy idea into a scoped PRD before any building starts. Two entry points: conversational chat or structured interview form.
/discovery   /interview
Takes In
  • An idea, problem statement, or existing doc (uploaded)
  • Optional reference docs to pre-fill answers
  • Mode selection (Quick / Thorough / Gap-Fill)
What Happens
  • Conversational chat across 6 stages: problem, scale, stakeholders, systems, success, risks
  • Or: structured interview form with section-by-section questions
  • Doc analysis fills fields automatically; conversation fills gaps
  • Agent generates triage summary or full PRD at the end
Outputs
  • Triage summary (Quick mode) — 1-page brief, ready for Build
  • Full PRD (Thorough / Gap-Fill) — scoped requirements doc
  • Brief text passed directly to the Build form
Discovery Modes
Quick (~5 min)Fuzzy idea → triage summary. No docs required.
Thorough (~15 min)Notes or draft → full PRD. More depth on each stage.
Gap-Fill (~10 min)Existing PRD or doc → completed PRD with gaps filled. Docs required.
2
KB Audit
Audit the knowledge base documents that will power the agent before spending time on a build. Catches gaps, conflicts, and quality problems early.
/kb-audit
Takes In
  • A named KB (set of documents to audit)
  • Uploaded documents or existing KB reference
What Happens
  • Scores each document across quality dimensions
  • Identifies conflicts between documents
  • Flags stale, incomplete, or ambiguous content
  • Generates repair recommendations per document
Outputs
  • Overall risk level (low / medium / high)
  • Per-document quality scores
  • Conflict map across docs
  • Prioritized repair list
3
Build
6-phase AI pipeline that turns a brief into a deployable Glean agent. Routes to one of four build modes based on what kind of output the brief calls for.
/build   /build-detail
Takes In
  • Brief text (from Discovery, Interview, or paste)
  • Model selection
  • Optional reference docs
  • Build mode (auto-detected or manual)
What Happens
  • Phase 0: Fitness check — is Glean the right platform?
  • Phase 1: Gap analysis — what's missing? HITL checkpoint.
  • Phase 2–3: Workflow design + instruction writing
  • Phase 4: Artifact generation (JSON, specs, etc.)
  • Phase 5: Summary + lessons learned
  • Post-build: JSON validation + budget audit
Outputs
  • agent-config.json (importable to Glean)
  • gap-analysis.md
  • phase-3-instructions.md
  • build-summary.md
  • SKILL.md (Skill builds)
  • OpenAPI spec (Action builds)
Build Modes
Agent BuildFull 6-phase pipeline → Glean Workflow JSON
Auto Mode BuildPhases 0, 1, 2A, 3A → Auto Mode brief (no JSON)
Skill BuildPhases 0, 1, then single Skill phase → SKILL.md
Custom ActionPhases 0, 1, then single Action phase → OpenAPI spec
HITL Checkpoint — After Phase 1, the gap analysis surfaces a verdict (PROCEED / CAUTION / BLOCK). You review and approve or cancel before the build continues. BLOCK requires a checkbox override.
4
Evaluate
Run structured tests against a built agent to measure accuracy, citation quality, and conversation behavior. Produces scored results that feed directly into Repair.
/evaluate
Takes In
  • Agent ID (selects which built agent to test)
  • Q&A CSV (for QA eval) — question + expected answer pairs
  • Scenario definitions (for Conversation eval)
What Happens
  • QA eval: sends each question to the agent, scores accuracy and citation quality per response
  • Conversation eval: runs multi-turn scenarios, scores each turn and overall session across 6 dimensions
  • Results aggregated with averages, failing questions flagged
Outputs
  • Per-question accuracy scores
  • Per-question citation quality scores
  • Conversation rubric scores (6 dimensions)
  • Failing questions list — input for Repair
  • Eval run stored with agent_id for history
Eval Modes
Q&A EvalUpload CSV of Q&A pairs. Scores accuracy + citation quality. Best for factual agents.
Conversation EvalMulti-turn scenario testing. 6 rubric dimensions: Goal Completion (25%), Process Quality (20%), Red Flag Detection (15%), Output Quality (15%), Downstream Readiness (15%), Conversational Skill (10%).
5
Repair
Takes eval failures as input and generates targeted instruction patches. Versioned — each repair run produces diffs you can review before applying.
/repair
Takes In
  • Eval run (from Evaluate) — the failing questions and scores
  • Agent ID — which agent's instructions to repair
  • Current phase-3-instructions.md from the build
What Happens
  • Analyzes patterns across all failing questions — not just individual fixes
  • Identifies which task instructions are responsible for failures
  • Generates specific diffs to those instructions
  • Each repair is versioned and stored with a changelog
Outputs
  • Repair diff — what changed and why
  • Updated instructions for affected tasks
  • Repair changelog entry
  • Ready to re-run Evaluate to measure improvement
Utility Tools (always accessible)
Validate
/validate
Checks agent-config.json against the Glean schema. Flags trigger config errors, toolConfig name mismatches, missing required fields, and sharingSettings nesting issues.
Budget
/budget
Audits instruction token count across all tasks. Shows per-task breakdown and total. Flags steps that exceed Glean's instruction budget limit.
Patterns
/patterns
Pattern library of reusable Glean agent designs. Includes Search + Think + Respond, Plan & Execute, branch-after-search patterns, and Tyler's anti-patterns.
Help
HelpPanel (sidebar)
In-app conversational help. Claude Haiku reads the full help corpus on every question. No retrieval — the whole corpus is loaded and prompt-cached. Persistent conversation history.
History
/ (home)
Activity timeline showing all builds, repairs, evals, discovery sessions, and KB audits. Status-filtered, with direct links to each artifact.
Settings / System
/settings   /system
Settings: model selection, API key configuration, team access. System: backend health, app version, current runtime info.
How the Steps Connect
From Hands off to What passes between them
1. Discovery 3. Build Brief text (triage summary or full PRD) pre-fills the Build form. User confirms and starts the build.
1. Discovery (Interview) 3. Build Interview answers assembled into brief text + reference docs, passed directly to the new build.
2. KB Audit 3. Build Indirect — you fix KB gaps before starting a build. No automated handoff; the audit informs document prep.
3. Build — Phase 0 Build — Phase 1 (or stops) FAIL verdict auto-stops the pipeline. PASS + mode recommendation (Agent vs Auto) routes the remaining phases.
3. Build — Phase 1 HITL Checkpoint Gap analysis with PROCEED / CAUTION / BLOCK verdict. Human approves before phases 2–5 run.
3. Build Validate + Budget agent-config.json automatically checked post-build. Results shown in the build detail view.
3. Build 4. Evaluate Agent ID assigned at build time. Evaluate uses it to target the right agent for testing.
4. Evaluate 5. Repair Eval run ID + failing question list. Repair loads these to scope what needs fixing.
5. Repair 4. Evaluate Updated instructions. Re-run eval against the patched agent to measure improvement. Iterate until scores stabilize.
